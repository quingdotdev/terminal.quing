# contributing

this project is early stage and focused on a strict multi project workflow. contributions are welcome when they align with the product scope and the slate design language.

## workflow

open an issue before large changes so the direction is agreed. use a feature branch for your work and keep pull requests small enough to review in one sitting. update docs when behavior changes and keep the ui consistent with the existing patterns.

## quality bar

| area | expectation |
| --- | --- |
| lint | `bun run lint` passes |
| build | `bun run build` passes |
| ui | no unreviewed visual regressions |
| behavior | no workspace data loss |
| docs | usage and architecture docs stay accurate |
| brand | Quing and slate rules are followed |

## review

reviews are direct and practical. if a change is too large or unclear, it will be sent back for revision without delay.
