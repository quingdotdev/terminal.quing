# docs map

this file explains the purpose of each document in this folder and when to use it.

| document | when to read it |
| --- | --- |
| overview.md | when you want the product intent and scope |
| usage.md | when you need keyboard shortcuts and operational detail |
| profiles.md | when you are changing shells, args, or cwd |
| workspace.md | when you are exporting, importing, or migrating state |
| architecture.md | when you need the runtime layout and data flow |
| development.md | when you are setting up a local dev environment |
| troubleshooting.md | when something is broken and you need a fix |
| docs-map.md | when you are not sure where to start |
| code-walkthrough.md | when you want a page by page code overview |
| brand.md | when you need the Quing brand rules |
| add-unsplit-show-split-tab-names-in-tab-strip.md | when you are implementing unsplit and split tab naming |
